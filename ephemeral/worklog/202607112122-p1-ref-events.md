## Goal

Implement roadmap phase 1: prove required Git ref transitions and derive
immutable pending push events plus queued authoritative workflow state from
accepted release-ref pushes.

## Decisions

- decision: Snapshot real branch refs immediately before and after
  `git-http-backend` receive-pack. Only actual ref differences create events.
- decision: Rejected pushes require no protocol parsing; an unchanged post-push
  ref map proves that no GitHub-side event or workflow state should be created.
- decision: Store an immutable raw event body at creation time so the later
  delivery layer can sign and redeliver the exact bytes.
- decision: Workflow configuration belongs to the private control plane. A
  configured release-ref movement creates a distinct queued run bound to its
  exact resulting SHA.
- correction: URL fields and internal domain data remain concrete values.
  Pointers remain limited to shared state, lifecycle/no-copy types, and external
  APIs requiring them.
